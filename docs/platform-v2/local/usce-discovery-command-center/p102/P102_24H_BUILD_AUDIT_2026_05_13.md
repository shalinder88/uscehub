## Bugs found and fixed during the period

1. **Scope-inference bug (P102-0E):** Generic medical-institution tokens ("hospital", "center", "medical", "health", "system", "university", "college", "group") were being treated as campus differentiators. Houston Methodist Hospital + houstonmethodist.org would have wrongly classified as HEALTH_SYSTEM_LEVEL because "hospital" wasn't in the domain. Fix: added GENERIC_INSTITUTION_TOKENS filter + tightened heuristic to require BOTH a positive specific-token domain match AND a missing campus-specific token.

2. **Identity domain match bug (P102-0W):** `domainMatches` used substring matching → 'hca' (substring of 'healthcare') wrongly matched HCA Healthcare registry entry against unrelated domains like 'stanfordhealthcare.org'. Fix: tightened to exact host or subdomain match.

3. **Identity system inference gap (P102-0W):** Required canonical name to include a known campus keyword. Failed for institutions whose canonical IS the system name (e.g., "Stanford Health Care" on stanfordhealthcare.org has no separate campus keyword). Fix: when domain strictly matches a system's domain, treat as system member regardless of name match.

All three bugs were found by tests added in the same sprints. No production impact — they were caught in test before they could affect real runs.
