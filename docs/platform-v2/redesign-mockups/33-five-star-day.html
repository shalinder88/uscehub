How verification actually runs

One pipeline. Five stages. No hidden algorithm.

01 · INGEST

Listing submitted

By an institution poster or imported from a curated source list.

02 · ADMIN REVIEW

Human admin reads copy

Approves or rejects based on accuracy. Source URL noted.

03 · CRON CHECK

Daily URL probe

02:00 UTC. HTTP status + content sniff. Pass / fail / unknown.

04 · QUEUE

Failures → admin

Within 24h, a human re-checks. Listing flagged "Needs recheck" until resolved.

05 · PUBLISH

Listing visible

With status badge + last-verified timestamp. Reviews kept structurally separate.
